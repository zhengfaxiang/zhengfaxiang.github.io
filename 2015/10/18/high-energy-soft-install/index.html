Install Softwares for High-energy Astronomy

Posted on 2015-10-18   |   In Astronomy   |  

DS9

Install

yaourt -S ds9-bin

Check Installation

Run DS9 by typing ds9 in your unix prompt

HEASOFT

Download source code

Visit http://heasarc.gsfc.nasa.gov/docs/software/lheasoft/download.html

Unpack the file to the installation directory

cd /usr/local/
sudo tar zxvf /path/to/the/downloaded/file/heasoft-6.17src.tar.gz

Install

cd heasoft-6.17/BUILD_DIR
sudo ./configure
sudo make
sudo make install

this may take some time, so be patient : -) or you can have a coffee ~

Add the environment variables

Add following code to your ~/.bashrc or ~/.zshrc

export HEADAS=/usr/local/heasoft-6.17/(PLATFORM)
alias heainit=". $HEADAS/headas-init.sh"

where (PLATFORM) is a placeholder for the platform-specific string
denoting your machine’s architecture, for example: x86_64-unknown-linux-gnu-libc2. 22
in my Archlinux

Check installation

Run heainit in your unix prompt

XMM SAS

Download

Download from http://xmm.esac.esa.int/sas/current/downloadandinstall.shtml

Unpack it to the installation directory

sudo mkdir /usr/local/SAS
cd /usr/local/SAS
sudo tar zxf /path/to/the/downloaded/file/sas_14.0.0-<OS>.tgz

Run the installation file

sudo ./install.sh

Environment settings

Add following code to your ~/.bashrc or ~/.zshrc

alias sasinit=". /usr/local/SAS/xmmsas_20141104_1833/setsas.sh"

Check installation

Run sasinit in your unix prompt

CALDB

Download

  • Chandra

    Visit http://cxc.harvard.edu/ciao/download/caldb.html#caldb to download Chandra caldb

  • Others

    Visit http://heasarc.gsfc.nasa.gov/docs/heasarc/caldb/caldb_supported_missions.html to download
    caldb of missions needed such as Suzaku

Unpack to the installation directory

mkdir ~/caldb
cd ~/caldb
tar zxvf /path/to/the/downloaded/file/caldb_4.6.9_main.tar.gz
tar zxvf /path/to/the/downloaded/file/goodfiles_suzaku_xis_bcf_20151005.tar.gz
tar zxvf /path/to/the/downloaded/file/goodfiles_suzaku_xis_cpf_20151005.tar.gz
tar zxvf /path/to/the/downloaded/file/goodfiles_suzaku_xis_index_20151005.tar.gz

Environment settings

Add following code to your ~/.bashrc or ~/.zshrc

export CALDB=$HOME/caldb
export CALDBCONFIG=$CALDB/software/tools/caldb.config
export CALDBALIAS=$CALDB/software/tools/alias_config.fits

CIAO

Before installation

link installed CALDB to your CIAO installation directory

ln -s ~/caldb ~/ciao-4.7/CALDB

Download ciao-install and run it

Download from http://cxc.harvard.edu/ciao/download/

./ciao-install

just install ciao to your home directory : -)

Install dependences

yaourt -S libselinux

and more…

Fix a bug

I got something like:

sh: symbol lookup error: sh: undefined symbol: rl_signal_event_hook

during installation, duckduckgoaround, I figured out following solution:

cd ~/ciao-4.7/ots/lib/
mv libreadline.so.6 libreadline.so.6.bak
mv libreadline.so libreadline.so.bak
ln -s /usr/lib/libreadline.so.6.3 ~/ciao-4.7/ots/lib/libreadline.so.6
ln -s /usr/lib/libreadline.so.6 ~/ciao-4.7/ots/lib/libreadline.so

Environment setting

Add following code to your ~/.bashrc or ~/.zshrc

alias ciao="source ~/ciao-4.7/bin/ciao.bash"

Check installation

ciao
check_ciao_caldb

ISIS

Ensure that slang has been installed into a standard location

Download source file using git

git clone http://space.mit.edu/cxc/isis/isis.git ~/Downloads/isis-src

Build and install

cd ~/Downloads/isis-src
./configure --with-headas=$HEADAS
make
make check
sudo make install

Check installation

After isis is installed, you should be able to run it by
typing isis at the unix prompt
